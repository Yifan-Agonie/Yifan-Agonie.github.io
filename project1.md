# Disaster Relief App  
Designing for fast, accessible support in emergency situations  

---

## 🧠 Overview

**Role:** UX Researcher & Designer  
**Duration:** 2024–2025  
**Context:** Hybytes (London)  
**Methods:** Contextual Inquiry, User Interviews, Usability Testing, Behavioural Analysis  

This project focused on improving how users register and request assistance during disaster situations, where time pressure, stress, and information overload significantly impact user behaviour.

---

## 🚨 Problem

In emergency scenarios, users need to act quickly under high cognitive load. However, existing systems presented several critical challenges:

- Complex and lengthy registration processes  
- High cognitive load and unclear task flows  
- Difficulty accessing relevant support quickly  
- Poor accessibility for multilingual and diverse users  

These issues reduced efficiency and delayed access to critical assistance.  [oai_citation:0‡Projects experence .pdf](sediment://file_0000000087d4720ba781535e2e3de0e8)  

---

## 🔍 Research

To understand real user needs in disaster contexts, I conducted:

### Contextual Inquiry  
Observed and analysed how users behave when applying for assistance in high-pressure situations  

### User Interviews  
Identified user expectations, pain points, and decision-making patterns  

### Usability Testing  
Evaluated whether redesigned flows improved task completion speed and clarity  

---

## 💡 Key Insights

Through qualitative and behavioural analysis, three key insights emerged:

### 1. Cognitive Overload  
Users struggled with long forms and excessive information under stress  

### 2. Inefficient User Flows  
Redundant steps and unclear navigation slowed down task completion  

### 3. Accessibility Gaps  
Language barriers and text-heavy interfaces reduced usability for diverse users  

These findings aligned with behavioural patterns observed in emergency scenarios.  [oai_citation:1‡Yifan.Li_UX research_CV_2026.pdf](sediment://file_00000000b520720b9db24352431ce9b7)  

---

## ✏️ Design Solutions

Based on research insights, I redesigned the registration and request experience:

### Simplified User Flow  
- Reduced redundant steps by ~20%  
- Streamlined task sequence for faster completion  

### Improved Information Structure  
- Reorganised content for clarity and prioritisation  
- Reduced cognitive load through clearer hierarchy  

### Multilingual & Accessible Design  
- Introduced multilingual support  
- Replaced long text with intuitive icons for faster understanding  

### Wireframing & Interaction Design  
- Designed detailed wireframes and flows in Figma  
- Defined interaction logic to support efficient user journeys  

---

## 🧪 Validation

The redesigned solution was tested through usability testing:

- Faster task completion  
- Improved clarity in user decision-making  
- Reduced confusion and navigation errors  

---

## 📈 Impact

- Improved usability and reduced friction in critical flows  
- Enabled faster access to emergency support  
- Enhanced accessibility for diverse user groups
